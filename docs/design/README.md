# docs/design

Purpose: stable design intent.

Belongs:

- design decisions
- boundaries and responsibilities
- invariants
- stable subsystem intent

Not allowed:

- exploratory brainstorming
- current work ledger status
- diary-style historical trace
- authoritative behavior contract text that belongs in spec

Design documents define stable intent, but they are not the work ledger.
