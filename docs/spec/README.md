# docs/spec

Purpose: authoritative behavior contract.

Belongs:

- implementation-facing requirements
- behavioral contracts
- testable properties
- specification text tied to executable or reviewable semantics

Not allowed:

- exploratory notes
- current work tracking
- diary-style historical narrative
- loose design brainstorming

Specifications define what implementation must satisfy.
